如何用grub24dos引导wubi安装的ubuntu

作者: LeonMa | 时间: 2014年9月1日

一、背景
出于尝鲜，使用wubi安装的方式在电脑上安装了ubuntu 10.04，在常用软件、字体设置等折腾得差不多顺手时，发现wps linux版和百度云的linux客户端bcloud等软件无法在ubuntu 10.04下安装，因此想安装ubuntu12.04。

在安装ubuntu12.04时发现要先卸载ubuntu 10.04才能继续，不想之前的折腾就这么丢失，因此思考着怎么能使多个wubi安装的ubuntu并存。之前用grub4dos成功引导过wubi安装的ubuntu9.04，套用老办法来引导ubuntu 10.04，却怎么也不成功，可能是从ubuntu10.04开始采用grub2引导的缘故把，无奈，只能另寻他法。
二、目的
1、用grub24dos引导wubi安装的ubuntu。
2、实现wubi安装的ubuntu的文件夹名字可以更改。
3、实现同时多个wubi安装的ubuntu并存，并分别能启动。
三、步骤
1、在win7下安装grub24dos

（1）下载grub24dos后将其解压到C:盘下，文件夹名为grub2；

（2）进入C:\grub2\install目录，然后运行grub24dos.exe（需要管理员权限） ；

（3）然后配置Grub2（不需要修改），点击OK（该步实际上是修改Windows BCD）；

（4）重新启动系统后，在Grub2的引导菜单中便能够见到Grub2 For Dos选项，然后就可以进入Grub2了；

下载地址：http://www.linuxidc.com/Linux/2012-07/66720.htm

2、用BCD引导的方式进入wubi安装的ubuntu系统，进入/boot/grub，打开grub.cfg中相应段落，拷贝到win7系统根目录下的\grub2的grub.cfg文件的start-user-section和end-user-sectio之间

下面是修改加入的引导ubuntu10.04的grub.cfg文件部分内容

### BEGIN /etc/grub.d/10_lupin ###
menuentry “Ubuntu 10.04, Linux 2.6.32-21-generic” {
insmod ntfs
set root=’(hd0,7)’
search –no-floppy –fs-uuid –set 5430907b309065b0
loopback loop0 /ubuntu1004/disks/root.disk
set root=(loop0)
linux /boot/vmlinuz-2.6.32-21-generic root=/dev/sda7 loop=/ubuntu1004/disks/root.disk ro quiet splash
initrd /boot/initrd.img-2.6.32-21-generic
}
menuentry “Ubuntu 10.04, Linux 2.6.32-21-gengrub.cfgeric (recovery mode)” {
menuentry “Ubuntu 10.04, Linux 2.6.32-21-generic” {
insmod ntfs
set root=’(hd0,7)’
search –no-floppy –fs-uuid –set 5430907b309065b0
loopback loop0 /ubuntu/disks/root.disk
set root=(loop0)
linux /boot/vmlinuz-2.6.32-21-generic root=/dev/sda7 loop=/ubuntu/disks/root.disk ro single
initrd /boot/initrd.img-2.6.32-21-generic
}
### END /etc/grub.d/10_lupin ###

3、保存，修改wubi安装的ubuntu文件夹名字，使之与grub.cfg文件一致，然后重启即可。
